## Commit 规范

Commit 规范参考的是 Angular 的 Commit 规范:

- feat: 一个新特性
- fix: 修了一个 Bug
- docs: 更新了文档（比如改了 Readme）
- style: 代码的样式美化，不涉及到功能修改（比如改了缩进）
- refactor: 一些代码结构上优化，既不是新特性也不是修 Bug（比如函数改个名字）
- perf: 优化了性能的代码改动
- test: 新增或者修改已有的测试代码
- chore: 跟仓库主要业务无关的构建/工程依赖/工具等功能改动（比如新增一个文档生成工具）
